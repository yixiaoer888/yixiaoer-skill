# 🎨 蚁小二技能文档标准化优化指令 (Unified Prompt)

任务目标：将指定的蚁小二技能原始 Markdown 文档重构为符合“WeChat Pay 标准”的高级交互式文档。

## 1. 核心规则

- **参数禁删原则**：严禁删除原始文档中的任何参数定义。所有字段、类型和说明必须在新的 Parameters 表格中全量体现。
- **三级结构对标**：必须包含以下五大标准板块：
    - **## 1. 触发场景 (Trigger)**：描述用户意图辨析、典型提示词。
    - **## 2. 交互协议 (Interactive Protocol)**：定义 Agent 执行时的逻辑规则（如字数校验、前置查询、资源上传原则）。
    - **## 3. 参数定义 (Parameters)**：使用标准 Markdown 对齐表格，包含“字段名、类型、必填、描述、默认值”。对复杂对象（如 raw 数据）需添加 3.1 复杂结构说明。
    - **## 4. 执行指令示例 (Command)**：提供一个完整的、包含 Action 的 JSON Payload 示例。
    - **## 5. 常见问题排查 (Troubleshooting)**：建立错误码/现象与解决方案的对照表。

## 2. 视觉标准

- **Alert 组件**：使用 GitHub/Google 风格：
    - `> [!IMPORTANT]` (前提条件)
    - `> [!TIP]` (技巧)
    - `> [!CAUTION]` (风险禁令)
- **文档标题**：统一格式为 `# 📄 [中文名称] [Type] 参数 ([English Name] [Type])`。

## 3. 技术性原则

- **数据完整性**：强调 raw 数据的“完整透传”和“先行查询”原则。
- **资源标识**：明确 `key` 是通过 `upload` 动作产生的唯一标识。

## 4. 调用指令格式

当用户或维护者要求优化文档时，Agent 应以此文档作为 Base Prompt，并将待处理文件内容按此结构重写。
