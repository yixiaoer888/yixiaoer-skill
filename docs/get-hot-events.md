# 📄 获取热点事件列表 Query 参数 (Get Hot Events Query)

获取平台实时的搜索热点、活动热点。用于在发布视频时绑定热门词汇、提升流量曝光。

> [!TIP]
> **流量密码**：绑定当前平台的实战热点是提升作品进入流量池概率的最快方式。Agent 应引导用户查询与其内容最匹配的热点。

## 1. 触发场景 (Trigger)

- **意图辨析**：用户希望蹭热点、追流量。
- **典型提示词**：
  - “看看现在抖音上有什么热门事件可以蹭”
  - “查询一下适合我这个美食视频的热点活动”

## 2. 交互协议 (Interactive Protocol)

1. **实时性要求**：此类数据变动极快，Agent 在每次发布前若用户有蹭热点意图，必须实时查询。
2. **相关性过滤**：Agent 应根据视频内容的语义，从返回的热点列表中筛选出最相关的前 3 个推荐给用户确认。
3. **数据完整性**：必须透传返回对象的整个 `raw` 结构到发布表单。

## 3. 参数定义 (Parameters)

| 字段名 | 类型 | 必填 | 默认值 | 描述 |
| :--- | :--- | :--- | :--- | :--- |
| **`action`** | `string` | **是** | `hot-events` | 固定操作码。 |
| **`account_id`** | `string` | **是** | - | 蚁小二账号 ID (`platformAccountId`)。 |

### 3.1 返回结果结构

| 字段 | 类型 | 描述 |
| :--- | :--- | :--- |
| `yixiaoerId` | `string` | 统一标识 ID |
| `yixiaoerName` | `string` | 热点事件名称 |
| `raw` | `object` | **核心透传对象**。 |

## 4. 执行指令示例 (Command)

```bash
node scripts/api.ts --payload='{"action":"hot-events","account_id":"67fb2f1735eeb3cf31db3d65"}'
```

## 5. 常见问题排查 (Troubleshooting)

| 现象 | 可能原因 | 处理建议 |
| :--- | :--- | :--- |
| **热点不显示** | 平台未返回动态热点或该账号无权限。 | 提示用户此类数据受平台实时策略影响。 |
| **发布时热点失效** | 热点具有时效性，Payload 构造时间太长。 | 建议在获取热点后立即完成发布指令构造。 |

---
> [!IMPORTANT]
> **合规建议**：严禁强制绑定与内容完全无关的点击量型热点，可能会导致账号被判为“内容不适”。
