# 📄 获取可挂载游戏 Query 参数 (Get Games Query)

获取在发布视频时可挂载推广的游戏列表。主要用于抖音等平台的“游戏推广”场景，通过挂载游戏链接实现内容深度联动或赚取推广收益。

> [!IMPORTANT]
> **资源透传原则**：在发布表单（如 `contentPublishForm.game`）中使用游戏信息时，**必须**完整保留并透传本接口返回的 `raw` 原始对象。

## 1. 触发场景 (Trigger)

- **意图辨析**：用户在发布游戏相关的短视频内容，并要求“挂载游戏链接”、“进行游戏推广”或“查找可用的小游戏”。
- **典型提示词**：
  - “我要推广这款小游戏，查看可用链接”
  - “帮我找一下抖音可挂载的游戏列表”

## 2. 交互协议 (Interactive Protocol)

1. **权限前置**：挂载游戏通常需要账号具有“游戏达人”或特定权限。Agent 应提示用户检查账号资质。
2. **搜索反馈**：支持通过 `keyword` 模糊搜索。Agent 在获取结果后应展示游戏名称及类型，确认后再进行 Payload 填充。

## 3. 参数定义 (Parameters)

| 字段名 | 类型 | 必填 | 默认值 | 描述 |
| :--- | :--- | :--- | :--- | :--- |
| **`action`** | `string` | **是** | `games` | 固定操作码。 |
| **`account_id`** | `string` | **是** | - | 蚁小二账号 ID (`platformAccountId`)。 |
| `keyword` | `string` | 否 | - | 游戏名称关键字。 |

### 3.1 返回结果结构

| 字段 | 类型 | 描述 |
| :--- | :--- | :--- |
| `yixiaoerId` | `string` | 统一游戏标识 |
| `yixiaoerName` | `string` | 游戏显示名称 |
| `raw` | `object` | **核心透传对象**。包含平台特定的 `game_id`, `anchor_id` 等原始参数。 |

## 4. 执行指令示例 (Command)

```bash
node scripts/api.ts --payload='{"action":"games","account_id":"67fb2f1735eeb3cf31db3d65","keyword":"王者荣耀"}'
```

## 5. 常见问题排查 (Troubleshooting)

| 现象 | 可能原因 | 处理建议 |
| :--- | :--- | :--- |
| **搜索不到指定游戏** | 该游戏不在平台的推广白名单内，或关键词输入有误。 | 尝试缩短关键词，或前往平台后台确认游戏状态。 |
| **挂载后链接不显示** | `raw` 数据不完整，或账号因违规被取消推广权限。 | 核实账号状态，并确保 Agent 已完整透传所有返回字段。 |

---
> [!TIP]
> **收益提示**：参与“游戏发行人计划”的账号挂载相关游戏后，可根据视频带来的安装量/活跃度获取收益。
