# 📄 获取挑战与话题列表 Query 参数 (Get Challenges Query)

获取平台当前进行的挑战活动或热门话题列表。这是在小红书、抖音等平台发布内容时增加曝光量、对齐趋势的必要前置操作。

> [!IMPORTANT]
> **资源透传原则**：在发布表单（如小红书的 `<topic>` 或抖音的 `topics`）中使用话题时，**必须**透传本接口返回的 `raw` 原始对象，否则会导致平台话题匹配失效。

## 1. 触发场景 (Trigger)

- **意图辨析**：用户希望在发布内容前找到合适的话题或活动进行挂载。
- **典型提示词**：
  - “帮我找找现在抖音上有什么热门话题”
  - “搜索一下小红书里关于‘春季穿搭’的挑战活动”
  - “查看这个账号能参加的活动列表”

## 2. 交互协议 (Interactive Protocol)

1. **账号前置**：必须先获取 `platformAccountId` (id)，因为不同账号可见的活动可能不同。
2. **关键词搜索**：若用户未指定关键词，Agent 可根据内容描述自动提取关键词进行搜索。
3. **列表交付**：展示话题名称及参与热度，引导用户选择 1-5 个核心话题。

## 3. 参数定义 (Parameters)

| 字段名 | 类型 | 必填 | 默认值 | 描述 |
| :--- | :--- | :--- | :--- | :--- |
| **`action`** | `string` | **是** | `challenges` | 固定值。 |
| **`account_id`** | `string` | **是** | - | 蚁小二账号 ID (`platformAccountId`)。 |
| `keyword` | `string` | 否 | - | 话题/挑战关键词。 |

### 3.1 返回结果结构

| 字段 | 类型 | 描述 |
| :--- | :--- | :--- |
| `yixiaoerId` | `string` | 蚁小二内部统一 ID |
| `yixiaoerName` | `string` | 话题/活动显示名称 |
| `raw` | `object` | **核心透传对象**。包含平台原始的 `id`, `name` 等数据。 |

## 4. 执行指令示例 (Command)

```bash
node scripts/api.ts --payload='{"action":"challenges","account_id":"67fb2f1735eeb3cf31db3d65","keyword":"美食"}'
```

## 5. 常见问题排查 (Troubleshooting)

| 现象 | 可能原因 | 处理建议 |
| :--- | :--- | :--- |
| **结果为空** | 关键词太冷门，或该平台不支持话题检索。 | 尝试使用更通用的词汇（如“穿搭” -> “时尚”）。 |
| **提示账号不在线** | 查询部分平台话题需要账号处于登录状态。 | 检查账号状态，引导用户刷新登录。 |
| **返回的话题无法点击/发布** | `raw` 数据不完整。 | 确保 Agent 完整获取了整个对象并正确填入发布 Payload。 |

---
> [!TIP]
> **曝光优化提示**：建议选择与内容高度相关且处于“热议”状态的話题，避免盲目堆砌不相关的挑战。
