# 📄 蚁小二多账号管理与分发助手

## 全局交互规范

> ‼️ **以下规则适用于本技能所有能力，优先级最高。**

1.  **账号状态先决条件**：执行发布 (`publish`) 或敏感查询前，必须先调用 `accounts` 确认目标账号 `status: 1`。若失效，立即引导重登。
2.  **分步确认协议 (Consent First)**：
    *   **理解意图** -> **同步策略** (告知 Agent 将读哪些文档) -> **征得同意** -> **参数收集** -> **模拟预览** -> **最终执行**。
3.  **零假设追问**：严禁推测用户意图。例如，用户说“帮我发一下”，必须追问：*“发布到哪个平台？使用哪个账号？发布视频还是文章？”*。
4.  **资源双路解耦**：所有非 key 资源必须经过 `upload` -> `material` (可选) 流程。

---

## 能力概览

1.  **矩阵环境查询** — 账号列表、状态监控、分类/话题/POI 等动态环境参数预检。
2.  **内容全能分发** — 图文/视频/文章一键同步，支持“蚁小二草稿”与“平台草稿”精准判别。
3.  **智能素材处理** — OSS 资源上传、素材库云同步、封面图自动化关联。
4.  **运营数据看板** — 实时监控发布日志、作品阅读量、账号粉丝增长等核心数据。
5.  **工业级故障诊断** — 基于 `errorCode` 的自愈式排障引导。

## 能力1：矩阵环境查询

> 用户要“查账号”、“找分类/位置”或检查发布环境时。

- **账号/分组** → [📄 账号列表查询.md](./docs/query-accounts.md)
- **环境参数 (分类/话题/地点)**：
    - 通用分类 → [📄 获取发布分类.md](./docs/get-publish-categories.md)
    - 抖音专项 (音乐/挑战/POI) → [📄 获取 POI 位置.md](./docs/get-locations.md) | [📄 话题挑战.md](./docs/get-challenges.md)

> **加载策略**：`SKILL.md` -> 确认平台 -> 加载环境文档。若账号失效，强制跳转能力 5。

---

## 能力2：内容全能分发

> 用户要“发内容”、“存草稿”或“同步作品”时。

- **‼️ 只检索、不生成**：严禁自行构造 JSON 结构。
- **发布路由**：
    - 文章类 → [📄 文章索引.md](./docs/publish/article/index.md)
    - 视频类 → [📄 视频索引.md](./docs/publish/video/index.md)
    - 图文类 → [📄 图文索引.md](./docs/publish/image-text/index.md)

> **加载策略**：`SKILL.md` -> 识别类型 -> 加载【内容索引】 -> 根据目标平台加载【平台详情说明】。
